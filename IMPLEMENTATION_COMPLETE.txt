═══════════════════════════════════════════════════════════════════
                    KEY DECISIONS
═══════════════════════════════════════════════════════════════════

✓ PYINSTALLER (not Docker)
  Reason: Simpler for local machines, no Docker installation needed
  Trade-off: Larger file (300 MB vs 100 MB for Docker)
  Benefit: Users don't need any additional software

✓ SINGLE EXECUTABLE (not multi-platform)
  Reason: Windows-only build is simpler and more reliable
  Trade-off: Doesn't work on macOS/Linux
  Future: Create separate build.sh for Unix-like systems

✓ EMBED PYTHON (not assume installed)
  Reason: Guarantees Python availability
  Trade-off: Larger file size
  Benefit: Users don't need to install Python

✓ PIN ALL DEPENDENCIES (not use ranges)
  Reason: Reproducible builds, no version conflicts
  Trade-off: Need to manually update versions
  Benefit: Same .exe behavior everywhere

✓ REMOVE TENSORFLOW (not work around)
  Reason: Incompatible with Python 3.12 on Windows
  Trade-off: Text CNN not available
  Benefit: Build actually completes successfully

✓ GRACEFUL FALLBACKS (not hard failures)
  Reason: App works even if models are missing
  Trade-off: Uses neutral predictions
  Benefit: Demo/testing possible without real models
